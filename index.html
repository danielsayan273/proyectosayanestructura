Conceptos Básicos

Las estructuras de datos son formas organizadas de almacenar y gestionar información para usarla de manera eficiente. En la programación orientada a objetos, permiten representar datos mediante clases y objetos, facilitando el acceso, la modificación y el manejo lógico de la información.

Clasificación

  • Lineales: Arreglos, Listas, Pilas, Colas.
  • No lineales: Árboles, Grafos.

Identificación

  • Pilas: LIFO (Último en entrar, primero en salir).
  • Colas: FIFO (Primero en entrar, primero en salir).
  • Árboles: Relación jerárquica entre nodos.

Recursividad

La recursividad es cuando una función se llama a sí misma para resolver un problema más pequeño.

function factorial(n) { if (n === 0) return 1; return n * factorial(n - 1); }

Tipos de Estructuras de Datos

Arreglos

Conjunto de elementos del mismo tipo almacenados en posiciones contiguas de memoria.

Pilas

Estructura LIFO donde el último elemento en entrar es el primero en salir.

Colas

Estructura FIFO donde el primer elemento en entrar es el primero en salir.

Listas Enlazadas

Conjunto de nodos que contienen datos y una referencia al siguiente nodo.

Árboles

Estructura jerárquica donde cada nodo puede tener hijos, formando ramas.

Recursividad

Técnica donde una función se llama a sí misma para resolver subproblemas.

Arreglos

Los arreglos son estructuras de datos lineales y estáticas que almacenan elementos del mismo tipo en posiciones contiguas de memoria.

  • Unidimensional: Vector simple. Ejemplo: [1, 2, 3, 4]
  • Bidimensional: Matriz. Ejemplo: [[1, 2], [3, 4]]
  • Tridimensional: Matriz de Matrices.Ejemplo [1][2][3]

Tipos de Arreglos
